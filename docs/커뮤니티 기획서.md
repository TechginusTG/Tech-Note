## 5. MVP (Minimum Viable Product)

### 5-1. 개념 설명
MVP란 **Minimum Viable Product**, 즉 **최소 기능 제품**을 뜻함.  
아이디어를 검증하고 사용자 반응을 빠르게 확인하기 위해,  
**꼭 필요한 최소한의 기능만 포함한 첫 번째 제품**을 개발하는 전략이다.

- **목적**
  1. 빠르게 사용자 반응을 확인한다.
  2. 리소스를 절약한다 (완성형을 만들다 실패하는 위험 줄임).
  3. 실제 사용자의 피드백을 바탕으로 개선·확장한다.

- **예시 비교**
  - 완성형 커뮤니티: 로그인, 알림, 채팅, 이벤트, 태그, 추천 시스템 등 수십 가지 기능 포함.
  - **MVP 단계**: 가장 필요한 로그인/계정 연동, 프로젝트 공유, 협업 기능만 제공 → 이걸로 실제 메이커들이 사용할 수 있고 가치를 검증 가능.

### 5-2. 이번 커뮤니티의 MVP 범위
- **GitHub 연동** → 메이커들이 프로젝트 저장소와 자연스럽게 연결 가능  
- **Google 계정 연동** → 편리한 회원가입 및 로그인  
- **프로젝트 갤러리** → 완성물/아이디어를 시각적으로 공유  
- **제작 일지 (Build Log)** → 작업 과정 기록 및 참고 자료 제공  
- **협업 보드 (이슈·할 일 관리)** → 공동 프로젝트 관리  

즉, **이 다섯 가지 기능**만 구현해도 “실제 사용자가 사용할 수 있는 최소한의 커뮤니티”가 완성된다.
